1.            Introduction

What does the component do?

Integration of OptCulture Extension – OC Rewards & CRM enables Magento stores to use OptCulture’s Loyalty and Reward programs features. OptCulture helps you maximize revenue per customer and new customer acquisition through 360-degree customer engagement. Details of the Loyalty and Reward program from OptCulture refer to OC Support.

Integration

Integration encompasses the deployment of a generic extension. The customer’s administrator may be required to modify OptCulture specific settings in their Magento store settings section.

2.            Extension Overview

Functional Overview

This extension allows Magento customers to install and use OptCulture’s loyalty and reward program on their store. Below is the expected flow of actions: 


 

Examples

The below examples are written from a user’s perspective who visits the retailer (Magento end-user) website.

·       The User visits the retailer website and gets 100 points for creating an account.

·       The User purchases on the website and can earn points for the purchase.

·       The User updates her birthday in the loyalty program and is awarded 100 points on her birthday.

·       User1 refers to a friend User2. User2 creates an account on the website and makes a purchase. User1 gets points for the purchase made by User2.

·       The User does not like the product and returns it. A credit memo is raised to refund the User. As soon as the credit memo is completed, the User’s purchase points are reverted.

·       User can earn reward points when they spend on special categories such as shopping at online stores, on specific items and so on.

Limitations, Constraints

OC Extension works with the default LUMA theme; the User can change the UI accordingly for their web page if needed.

Compatibility

Available since Magento 2.4.x.

Privacy, Payment

The customer’s credit card data is not accessed. OptCulture only accesses the following info:

  1. Customer Email, Customer First Name, and Customer Last Name.

  2. Details of the order placed by the User on the customer’s Magento store.

  3. Details of the orders returned by the User on the customer’s Magento Store.

3.            Implementation Guide

Installation

Here we will guide you through the installation process of our extension. First off, open FileZilla, log in to your host as follows:

 

 

We switched the local site to the extension directory and the remote site to the code directory in the above example. Now on the remote site window, open the app/code directory, and on your local site window, right-click on Extension Name and click Upload. The extension will be uploaded onto your host in a few minutes.

The second method will require you to compress the extension into .zip or .tar.gz format, then execute the following command in your Command-Line/Terminal:

scp <path to your extension> <ssh server>:<path to app/code directory>

 

Then go to the app/code directory and execute the following command to decompress the module:

unzip <.zip compressed extension>

Or

tar -xzvf <.tar.gz compressed extension>

 

Now go to your Magento root directory, say it’s named magento2, and execute the following commands:

php bin/magento setup:upgrade

 

After running all these, your Command line/Terminal window might look like this:

 

 

Finally, you need to flush the cache by either type this into the Command Line/Terminal:

php bin/magento cache:flush

 

Now you can go to your store and check if the extension is working as expected.

Authentication Method

The OptCulture Extension provides three types of Authentication Methods:

  1. None

  2. Verification Code

  3. Check with signed-in details

None

Select None for no authentication.

Verification Code

Select Verification Code option for authentication through email or SMS during enrollment, redemption and promotions.

Case 1

When the customer tries to redeem his balance on the Cart page, he/she needs to enter the redeem balance value at the Redeemable Balance slot and click Apply.

 

Customer needs to confirm the verification code received on Email or SMS by entering it in the Verification Code section:

 

 

Customers can also see the expiration time of the verification code above the submit button.

Generate a new verification code by clicking on the RESEND CODE button when it expires.

 

 

Case 2

If a guest user tries to redeem the balance, he/she must sign in with the registered Email and Password.

 

User needs to confirm the verification code received on the registered Email or SMS by entering it in the Verification Code section:

Check with signed-in details

Select the Check with signed-in details option for checking the user email address or phone number entered during redemption and apply any promotion rules with the Magento signed-in user details. If not signed in, a prompt showing the details are not matched and the redemption will be failed.

Case 1

If a guest user tries to redeem the balance, he/she must sign in with the registered Email and Password.

 

Case 2

If an unauthorized user wants to avail redeem points, a notification appears stating, “Your signed in email address is not valid. Please sign in with the appropriate account to redeem”.
